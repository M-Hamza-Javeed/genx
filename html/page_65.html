Here, we first defined a template where we kept a place for the employee list

to go. Then we mentioned the itemView option as the EmployeeView class that

will be used by the collection to create an instance and populate it with each

employee data. These item views will be stacked in the element mentioned in the

itemViewContainer property. Now, let's create the composite view instance, add

a company to it, and render it, as shown in the following code snippet:

var companyView = new CompanyView();

// Add a company details

companyView.addCompany({

name: 'Innofied',

specialty: 'Team of JavaScript specialists',

employees: [{

name: 'Swarnendu De'

}, {

name: 'Sandip Saha'

}]

});

$(document.body).append(companyView.render().el);

So, you can see that a composite view provides a compact mechanism to display a

model and a collection associated to it in a single view. You may have a tree structure

data, and for that, multiple composite views need to be created. By default, the

rendering mechanism of a composite view is hierarchical in nature and the itemView

property is of the CompositeView type, if not overridden.
